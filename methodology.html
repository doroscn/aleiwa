Data Methodology

This page explains how Aleiwa collects, validates, and presents network-related data, including ASN rankings and public DNS server availability.

Data Sources

Aleiwa aggregates network data from multiple public routing datasets, IP allocation records, and active measurement results. Country mappings are based on ISO 3166 standards.

DNS Server Testing

Public DNS servers are actively tested for availability and responsiveness. Each server is queried periodically to verify that it responds correctly to standard DNS requests.

Servers that fail to respond, return invalid answers, or become unreachable are marked as unavailable.

ASN Ranking Logic

ASN rankings are derived from observed IP address distribution and routing visibility within each country. Percentages represent relative presence rather than exact user counts.

Update Frequency

Data is refreshed on a regular basis to reflect changes in network topology, routing behavior, and DNS availability. Update timestamps are displayed on individual pages where applicable.

Limitations

Network measurements are subject to change and may not represent real-time conditions. Results should be used for reference and analysis purposes rather than as absolute metrics.
